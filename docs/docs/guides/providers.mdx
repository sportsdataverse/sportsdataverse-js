# Provider recipes

Beyond ESPN, `sportsdataverse` wraps five standalone provider families. Each is
its own top-level namespace (no `{sport}`/`{league}` nesting) and every method
honors `{ parsed: true }`:

| Namespace | Provider | Notes |
|---|---|---|
| `sdv.odds` | The Odds API | needs **your** API key (`api_key`) |
| `sdv.cbs` | CBS Sports NAPI | public, anonymous; league/team ids are slugs |
| `sdv.fox` | Fox Sports Bifrost | public apikey defaults out of the box |
| `sdv.yahoo` | Yahoo shangrila / editorial | pass a `league` slug |
| `sdv.recruiting` | 247Sports | recruiting rankings, transfers |

## Odds — The Odds API

The Odds API requires a key (free tier at the-odds-api.com). Supply it via
`api_key`. The `sports` endpoint lists every league it covers:

```js
import sdv from 'sportsdataverse';

const sports = await sdv.odds.oddsApiSports({
  api_key: process.env.ODDS_API_KEY,
  parsed: true,
});
console.table(sports);
```

[Open in playground ▶](/playground?l=odds&e=flat%3Aodds_api%3Asports&parsed=1&api_key=YOUR_ODDS_API_KEY)

Live and inline — paste a real key into the `api_key` field first (calls without
one return 401):

:::

## CBS — league metadata

CBS league ids are slugs like `football-nfl`, `basketball-nba`, `baseball-mlb`.

```js
import sdv from 'sportsdataverse';

const league = await sdv.cbs.cbsNapiLeague({ league_id: 'football-nfl', parsed: true });
console.table(league);
```

Live and inline — the `league_id` is a CBS slug (`football-nfl`,

## Fox — Bifrost scoreboard

The public `apikey` + `api-version` pair default automatically — just pass a
`sport` (`cfb`, `nfl`, `mlb`, `nba`, …).

```js
import sdv from 'sportsdataverse';

const board = await sdv.fox.foxBifrostScoreboard({ sport: 'cfb', parsed: true });
console.table(board);
```

Live and inline — the public `apikey` defaults out of the box; just pick a
`sport` (`cfb`, `nfl`, `mlb`, `nba`, …):

## Yahoo — league standings

Yahoo's shangrila stats-graph. Pass a `league` slug (`ncaaf`, `nfl`, `nba`, …).
Some Yahoo hosts want an `Origin`/`Referer` — pass them via `headers` if a call
is rejected.

```js
import sdv from 'sportsdataverse';

const standings = await sdv.yahoo.yahooShangrilaLeagueStandings({
  league: 'ncaaf',
  parsed: true,
});
console.table(standings);
```

Live and inline — `lang`/`region`/`tz` default automatically; pass a `league`
slug. If a Yahoo host rejects the call, add an `Origin`/`Referer` via `headers`:

  endpoint="flat:yahoo_shangrila:league_standings"

## 247Sports — recruiting rankings

Scope with `year` + `sport_key` (`football`, `basketball`).

```js
import sdv from 'sportsdataverse';

const rankings = await sdv.recruiting.sports247Rankings({
  year: 2025,
  sport_key: 'football',
  parsed: true,
});
console.table(rankings);
```
